# Phase 2：个股、股指期货、期权扩展方案

本阶段目标是在 Phase 1 指数代理穷尽基线之外，加入更接近附文收益目标的高弹性资产与衍生品模块。结论仍以仿真实验证据为准，不把任何策略包装成收益承诺。

## 1. 全 A 个股线

第一批策略族：

- S2：全 A 横截面强势股动量。
- S3：趋势突破 + ATR / 均线止损。
- S4：小盘 / 微盘 / 流动性风险溢价。

必须先补齐的数据表：

- 交易日历。
- 股票基础表：代码、名称、交易所、上市日期、退市日期、上市状态、板块。
- 日频行情：open/high/low/close/volume/amount/turnover/pre_close。
- 复权因子或复权价：信号使用复权价，撮合使用真实价。
- ST、停牌、退市状态。
- 涨跌停价：至少要有 limit_up / limit_down。
- 如果使用指数成分，必须使用历史成分；当前成分股回填不能进入真实排行榜。

第一版过滤：

- 排除 ST、退市、停牌、上市未满 120 个交易日。
- 排除价格低于 2 或 3 元的股票。
- 排除 20 日平均成交额低于 5000 万 / 1 亿 / 3 亿的股票，做三档敏感性。
- 单票成交不超过当日成交额 5% / 10%。

第一版撮合：

- T+1。
- 涨停买入成交率：0%、20%、50%。
- 跌停卖出成交率：0%、10%、30%。
- 卖不出时记录 liquidity_trap，并继续持仓。

## 2. 股指期货 overlay 线

第一批合约：

- IF：沪深300股指期货。
- IH：上证50股指期货。
- IC：中证500股指期货。
- IM：中证1000股指期货。

建模关键：

- 合约价值 = 指数点位 * 合约乘数。
- 保证金 = 合约价值 * 保证金率 * 期货公司上浮系数。
- 资金不足时不能开仓；不能用小资金买“0.1 手”真实期货。
- 每日盯市，亏损从现金扣除。
- 维持保证金不足时触发减仓或强平。
- rollover 先用月度移仓近似，再引入真实主力合约和基差。

早期资金约束非常关键。IF/IH 按每点 300 元建模，IC/IM 按每点 200 元建模；最低保证金率先用合约价值 8% 作为交易所下限，主结果使用 12% / 15% / 20% 三档有效保证金。以指数 5000 点估算，一手名义本金可能在 100 万以上，即使用最低保证金，实盘还要有保证金缓冲。月入金 3 万、前期资产几十万时，很多窗口应直接判定为“买不起一手”。

第一版策略：

- beta overlay：在指数动量为正时尝试增加 +0.3 / +0.5 / +1.0 目标 beta，但按整手合约离散化。
- crash hedge：指数跌破 MA200 时尝试空头对冲，但也按整手合约和保证金约束执行。
- buffer：保证金需求必须小于现金的 50% / 33% / 25%，对应 2x / 3x / 4x buffer。

## 3. 股指/ETF 期权 overlay 线

第一版不先伪装成完整期权链回测，而做两层：

### 3.1 期权近似层

使用 Black-Scholes / Black-76 近似，输入为：

- 标的指数或 ETF 价格。
- 30/60/90 日到期。
- 25/35/50/65 delta。
- 历史波动率或简化 IV 分位。
- 权利金预算：NAV 的 0.5%、1%、2%、3%、5%。

输出：

- 买 call、牛市价差、保护性 put 的近似 PnL。
- IV crush 压力：隐含波动率下降 30%。
- 到期归零、止盈、止损规则。

这层只回答“凸性预算有没有理论上抬高达标概率”，不能回答“真实合约是否买得到、买卖价差如何”。

### 3.2 真实期权链层

等拿到真实期权链后再做：

- 上交所 / 深交所 ETF 期权每日合约。
- 中金所股指期权每日合约。
- bid/ask、成交量、持仓量。
- 到期、行权价、合约单位、熔断和涨跌幅。

第一版只做买权利仓和保护性买入，不把裸卖期权当安全增强。

ETF 期权先按常见 10000 份合约单位建模；中金所股指期权按每点 100 元合约乘数建模。真实链回测时必须按每日合约资料、调整公告、bid/ask、成交量、持仓量和到期规则逐日核对。

参考口径：

- 中金所合约条款：IF/IH 每点 300 元，IC/IM 每点 200 元，股指期权每点 100 元，最低保证金率需按合约和日期核对。
- 上交所 ETF 期权合约条款：ETF 期权合约单位常见为 10000 份，真实回测仍需按每日合约资料和调整公告核对。

## 4. 执行顺序

1. 保留 Phase 1 指数穷尽基线，作为收益目标难度下限。
2. 先做全 A 个股日频数据闭环和 S2/S3/S4，最可能真正提高达标概率。
3. 再加股指期货 overlay，但严格执行整手合约和保证金约束。
4. 再加期权近似层，验证凸性预算是否值得拉真实期权链。
5. 最后才做真实期权链和打板/连板模型。

当前仓库已经具备 Phase 2 的入口骨架：

- `scripts/build_phase2_realdata.py`：真实 raw 表到 `processed/phase2/stock_panel.parquet`。
- `src/quant_proof/realdata/`：交易日历、股票池、复权信号价、涨跌停、停牌、ST 和退市状态。
- `src/quant_proof/engine/`：T+1、停牌、涨跌停、费用、印花税和成交金额上限。
- `src/quant_proof/real_strategies.py`：S2/S3/S4 的真实个股候选配置与信号打分。

这些入口不代表真实 leaderboard 已可运行；必须先通过 `scripts/validate_phase2_real_data.py` 的数据门禁。

## 5. Phase 2 成功门槛

进入下一阶段的策略至少需要：

- 历史滚动 `P_success >= 5%`，否则只是偶然窗口。
- bootstrap `P_success >= 3%`。
- `P(max_drawdown > 50%)` 不应失控。
- 个股策略在低成交率 / 跌停卖不出压力下仍能保持非零通过率。
- 期货策略不能主要依赖“买不起一手时假装持有分数手”。
- 期权策略必须扣除权利金损耗，并做 IV crush 压力测试。
